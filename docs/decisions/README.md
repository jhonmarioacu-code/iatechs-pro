# Architecture Decisions (ADR)

Fecha: 2026-06-26  
Estado: Oficial

## Objetivo

Registrar decisiones tecnicas de alto impacto para mantener coherencia arquitectonica en el tiempo.

## Cuando crear una ADR

Crear ADR cuando se cambie o afecte:

- Stack base (runtime, framework, database, cache, colas)
- Estrategia multi-tenant
- Estructura de dominios o patrones obligatorios
- Contratos API de alto impacto
- Integraciones estrategicas (ejemplo: IA empresarial)

## Formato recomendado

1. Contexto
2. Decision
3. Consecuencias
4. Estado (propuesta/aprobada/sustituida)
5. Fecha
